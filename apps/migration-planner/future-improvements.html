Migration Plan/Future Improvements

Future Improvements

Features not currently in Odoo — to be built as needed
These are new capabilities that don't exist in Odoo today. They can be built in SERP/Laravel independently of the core migration phases, prioritized by business need. Each feature writes back to Odoo where necessary but the logic lives in SERP.

Why These Are Separate

These features don't exist in Odoo. The main migration phases focus on replacing existing Odoo functionality with better SERP interfaces. These improvements are net-new capabilities that can be built on-demand when the business needs them.

Available Improvements

Blanket PO Tracking

Track long-term vendor agreements with multiple releases. SERP manages the agreement, Odoo receives individual release POs.

How it works:
  • Create Agreement: User creates blanket PO in SERP with total qty, price terms, validity period — stored in Laravel
  • Release Orders: User creates releases against the blanket — each release creates a real purchase.order in Odoo
  • Track Usage: SERP tracks total released vs. agreement amount, shows remaining balance
  • Receipts: Individual PO receipts flow through normal receiving process
Why not in Odoo: Odoo has no native blanket/framework PO support. No purchase_requisition or similar tables exist. This is a feature gap that must be built custom.

Approval Workflows

Route POs for approval based on amount thresholds. Notify via Slack, update Odoo when approved.

How it works:
  • Threshold Rules: Configure in SERP: POs over $X need manager approval, over $Y need director
  • Multi-Step Approval: Ops approval first, then Finance approval for larger amounts
  • Slack Notifications: Approver receives Slack message with approve/reject buttons
  • Approve: Approver clicks approve in Slack — SERP updates PO status — calls Odoo to confirm purchase.order
  • Reject: Approver rejects — SERP notifies requestor, PO stays draft in Odoo
Why not in Odoo: Odoo has studio_approval_rule table but 0 active rules configured. Current workflow is state-based only (draft→purchase) with no enforcement of who can approve. Multi-step ops→finance approval doesn't exist.

Supplier Forecast → Odoo Integration

Turn SERP supplier forecasts into draft POs in Odoo with one click.

How it works:
  • Generate Forecast: Supplier Forecast tool calculates what to order based on demand
  • Review in SERP: User reviews suggested quantities, adjusts as needed
  • Create Draft POs: User clicks "Send to Odoo" — SERP calls Odoo to create purchase.order records in draft state
  • Finalize: POs can be confirmed through normal approval workflow
Why not in Odoo: Odoo has no forecasting integration. The Supplier Forecast tool already exists in SERP — this adds the ability to push results directly to Odoo as draft POs.

Inventory Adjustment Approvals

Require approval for manual inventory adjustments above a threshold. Track who approved and why.

How it works:
  • Adjustment Request: User submits inventory adjustment with reason code
  • Threshold Check: Adjustments above configured threshold require approval
  • Slack Notification: Approver notified via Slack with adjustment details
  • Audit Trail: All adjustments logged with who requested, who approved, and why
Why not in Odoo: Odoo allows direct inventory adjustments with no approval gate. This adds governance for high-value inventory changes.

Other Potential Improvements

Additional features discussed but not yet prioritized:

  • Bulk BOM Import: Import BOMs from spreadsheet with validation
  • Product/BOM Archiving: Soft-delete with archive flag instead of hard delete
